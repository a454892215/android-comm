从项目框架搭建来说 你说的还是太复杂了，主要可以分为4个模块:
   1.第一个模块是项目主体配置模块
     ✅ 全局异常处理（UI / async / 平台异常兜底）
     ✅ UI 适配方案（屏幕、字体、比例）
     ✅ 全局 UI 行为配置:android zhuang Loading DialogToast 字体缩放
     ✅ 路由体系配置：路由列表 初始路由 404 兜底 路由监听
   2.第二是网络请求模块
     统一http网络请求入口，方便进行统一的http请求配置
     统一处理请求参数，传承数据封装，数据加密等
     统一对相应数据进行解密，封装
     统网络异常处理和错误码处理
     统一网络请求log输出
     Token、登录态管理

   3.通用UI组件模块
     通用按钮
     通用弹窗
     空页面 / 错误页
     Loading / Skeleton

   4. 第四是常用工具类模块。
      Log工具
      时间日前 / 格式化工具类
      本地存储工具类
      权限工具类
      设备信息

 为什么要搭建框架？
 1. 节省时间提高新项目开发效率
 2. 把过去项目碰到的坑，在项目通用框架层统一处理，以免后续新项目再遇到
 3. 把过去很成熟的通用程度很高的代码模块抽抽取到项目通用框架层，以大幅度提高效率 避免不必要的bug
