MySQL 的字段类型非常丰富，为了方便记忆和开发，我们通常将其分为 **数值、字符串、日期时间、二进制** 以及 **特殊类型** 五大类。
### 1. 数值类型 (Numeric Types)
| 类型 | 大小 (字节) | 常用场景 | Java 对应类型 |
| --- | --- | --- | --- |
| **TINYINT** | 1 | 状态(0/1)、布尔值、小年龄 | `Integer` / `Byte` |
| **SMALLINT** | 2 | 排序、身高、几千内的数量 | `Integer` / `Short` |
| **MEDIUMINT** | 3 | 中等数值（如神话人物寿命） | `Integer` |
| **INT** | 4 | 用户ID、普通计数 | `Integer` |
| **BIGINT** | 8 | 雪花算法ID、总金额(分)、大流量计数 | `Long` |
| **DECIMAL** | 变长 | **精密货币金额** (如 10.25) | `BigDecimal` |

> **注意：** 存储钱千万不要用 `FLOAT` 或 `DOUBLE`，会有精度丢失风险。

---

### 2. 字符串类型 (String Types)
字符串的选择核心在于“定长”还是“变长”。
| 类型 | 最大长度 | 特点 | 常用场景 |
| --- | --- | --- | --- |
| **CHAR** | 255 字符 | 定长，速度快，末尾补空格 | 手机号、身份证、MD5加密后的密码 |
| **VARCHAR** | 65535 字节 | 变长，节省空间 | 姓名、地址、简介、邮箱 |
| **TEXT** | 64 KB | 长文本，不占行存储限制 | 文章内容、备注、公告 |
| **LONGTEXT** | 4 GB | 极长文本 | 电子书、HTML源码 |

---

### 3. 日期与时间类型 (Date and Time Types)
| 类型 | 格式 | 范围 | 特点 |
| --- | --- | --- | --- |
| **DATE** | YYYY-MM-DD | 1000~9999年 | 生日、纪念日 |
| **DATETIME** | YYYY-MM-DD HH:MM:SS | 1000~9999年 | **推荐：** 绝对时间，不受时区影响 |
| **TIMESTAMP** | YYYY-MM-DD HH:MM:SS | 1970~2038年 | 受时区影响，会自动随系统时区变化 |
| **TIME** | HH:MM:SS | -838~838小时 | 营业时间、耗时时长 |

---

### 4. 二进制类型 (Binary Types)

通常用于存储文件原始数据，但 Java 后端一般**不建议**直接把文件存数据库。
* **BINARY / VARBINARY**：定长/变长二进制字符串（类似 CHAR/VARCHAR，但不带字符集）。
* **BLOB (Binary Large Object)**：存储图片、音频、视频的二进制流（生产环境建议存阿里云 OSS 等，数据库仅存 URL 字符串）。
---

### 5. 特殊类型
* **ENUM (枚举)**：限制字段只能选预设的值（如 `'male','female'`）。虽然省空间，但扩展性差，Java 后端通常在代码里维护枚举，数据库用 `TINYINT`。
* **JSON**：MySQL 5.7+ 支持，可以直接存储 JSON 对象并支持通过 `->` 语法查询内部属性。非常适合配置信息、动态扩展字段。
--

### 选型口诀：
1. **能小不大小**：能用 `TINYINT` 就不用 `INT`。
2. **定长用 CHAR**：如手机号、状态码。
3. **钱用 DECIMAL**：禁止使用浮点数。
4. **时间用 DATETIME**：避开 2038 年的坑。
5. **字段必带 COMMENT**：为了你自己和同事的头发。
