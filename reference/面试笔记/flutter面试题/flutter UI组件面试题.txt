
## **1️⃣ Container 的“包裹内容”能力是被动的**
* **Container 并不能自主决定自己的大小**
* 它的尺寸完全依赖于 **父 Widget 的约束（constraints）**

## **2️⃣ 不同父约束的结果**
| 父约束类型                                          | Container 行为                                                       |
| ---------------------------------------------- | ------------------------------------------------------------------ |
| **无限约束（例如 Row/Column 主轴方向）**                   | Container 尝试尽量小，但会受子 Widget layout 限制；复杂子 Widget 可能撑满 → 不能保证完全包裹内容 |
| **固定尺寸（SizedBox/ConstrainedBox 固定宽高）**         | Container 尺寸被固定 → 无法按内容收缩                                          |
| **最大约束有限（BoxConstraints(maxWidth/maxHeight)）** | Container 可以包裹内容，但不会超过父最大约束                                        |
| **无约束 / 可伸缩父容器**                               | Container 可以完美包裹内容 + padding                                       |

> 总结一句话：**Container 能否包裹内容，完全取决于父 Widget 给的约束，而不是 Container 自主决定。**
