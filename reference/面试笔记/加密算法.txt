严格来说，你理解的核心分类是对的：绝大部分软件开发用到的加密机制，核心上可以归为三类：

1. **哈希算法（Hash / 摘要）**

   * 功能：单向不可逆，把任意长度数据变成固定长度摘要。
   * 示例：MD5、SHA-1、SHA-256、SHA-3
   * 特点：不可解密，只能验证数据完整性或生成签名。

2. **对称加密（Symmetric Encryption）**

   * 功能：用同一把密钥加密和解密数据。
   * 示例：AES、DES、3DES、Blowfish
   * 特点：加密解密快，但密钥管理难。

3. **非对称加密（Asymmetric Encryption）**

   * 功能：用公钥加密、私钥解密（或私钥签名、公钥验签）。
   * 示例：RSA、ECC、DSA
   * 特点：加密速度慢，但可安全传输密钥、做数字签名。

---

> 其他像 **HMAC / MAC**、**混合加密** 等，其实都是**对称或非对称加密 + 哈希**的组合或应用场景上的扩展，并不是独立的底层加密类别。

如果你只想记核心底层算法，确实就是：**哈希 / 对称 / 非对称** 三类即可。

HTTPS 并不是单独用某一类加密算法，而是 组合使用三类算法：
好的，我们来详细梳理 HTTPS（即 HTTP over TLS/SSL）协议中使用的加密算法及其作用。HTTPS 并不是单独依赖一种算法，而是一个**混合加密系统**，它结合了对称加密、非对称加密和哈希算法来保证通信的**安全性、完整性和身份认证**。下面分层次详细说明：

---

## **一、非对称加密（Asymmetric Encryption）**

### **作用**

1. **密钥交换**：HTTPS 会在握手阶段使用非对称加密来安全地交换对称加密所用的会话密钥（Session Key）。
2. **身份认证**：通过证书（Certificate）验证服务器身份，防止中间人攻击（MITM）。

### **常用算法**

* **RSA**：最常见，用于加密对称密钥或数字签名验证。
* **ECC（椭圆曲线加密）**：密钥更短、效率更高，现代 TLS 越来越多使用。

### **举例**

客户端获取服务器公钥 → 用公钥加密生成的随机对称密钥 → 服务器用私钥解密得到对称密钥 → 双方拥有共享会话密钥。

---

## **二、对称加密（Symmetric Encryption）**

### **作用**

1. **数据加密**：在握手阶段生成的对称密钥用于加密实际传输的数据，保证数据机密性。
2. **高效率**：对称加密比非对称加密快得多，适合大量数据传输。

### **常用算法**

* **AES（Advanced Encryption Standard）**：最常用，支持 128/192/256 位密钥。
* **ChaCha20**：高性能加密算法，尤其在移动端和低性能设备上优于 AES。

### **模式**

* **CBC（Cipher Block Chaining）**
* **GCM（Galois/Counter Mode）**：现代 TLS 推荐，支持加密 + 完整性校验（AEAD 模式）。

---

## **三、哈希算法 / 消息认证码（Hash / MAC）**

### **作用**

1. **完整性校验**：保证传输的数据未被篡改。
2. **数字签名**：在握手中用于验证服务器证书签名的真实性。
3. **HMAC（Keyed-Hash Message Authentication Code）**：在 TLS 中结合对称密钥进行消息验证，确保数据的完整性和来源可靠。

### **常用算法**

* **SHA-1**：早期 TLS 使用，安全性已不足。
* **SHA-2（SHA-256、SHA-384）**：现代 TLS 推荐。
* **SHA-3**：新一代哈希标准，部分新系统可能使用。
* **HMAC-SHA256 / HMAC-SHA384**：用于消息认证码。

---

## **四、HTTPS 加密流程总结**
HTTPS 实际加密流程可分为握手阶段和数据传输阶段：
1. **握手阶段（TLS Handshake）**
   * 客户端发起连接 → 请求服务器证书
   * 服务器发送证书 → 客户端验证证书（非对称加密 + 哈希验证签名）
   * 客户端生成对称会话密钥 → 用服务器公钥加密 → 发送给服务器
   * 服务器用私钥解密 → 双方共享对称密钥

2. **数据传输阶段**
   * 使用对称加密算法（AES/ChaCha20）加密数据
   * 使用 HMAC/SHA 对数据生成校验码 → 保证完整性
   * 传输过程中即保证机密性、完整性和防篡改
---

### **五、HTTPS 使用算法的作用总结表**
| 类别       | 具体算法                | 作用            |
| -------- | ------------------- | ------------- |
| 非对称加密    | RSA、ECC             | 密钥交换、身份认证     |
| 对称加密     | AES、ChaCha20        | 加密传输数据，保证机密性  |
| 哈希 / MAC | SHA-256、HMAC-SHA256 | 数据完整性校验、防篡改   |
| 数字签名     | RSA / ECC + 哈希      | 验证服务器身份、证书合法性 |
---
**关键点记忆**：
1. 非对称加密 → 握手密钥，慢但安全
2. 对称加密 → 数据传输，快且高效
3. 哈希 / HMAC → 验证完整性
4. HTTPS = 三者组合 → 安全、完整、可信
