## 1️⃣ Dart 是强类型还是弱类型？是静态还是动态？
  强类型， 静态类型（但支持类型推断）

📌 Dart 在 **编译期就做类型检查**（AOT/JIT 都是）

`var`、`dynamic`、`Object` 的区别？
`var`     类型推断关键字，用于声明一个变量，编译后，会根据变量第一次赋值的类型，自动推断，然后固定变量的类型
`Object`  所有类的基类，所以可以声明任何非空类型的变量（它是 Dart 类层次结构的根节点）
`dynamic` 变量声明关键字 是动态类型 编译期间不会检查 可以赋值任何类型

dart中声明变量的所有方式：
typedef  类型别名,用于给已经存在的类型定义一个新的名字。在dart中的作用是给复杂的类型(包含类型内部泛型约束部分)或者函数起别名：
         函数也是一种类型，函数的类型由返回类型和参数共同决定。
  1. typedef UserList = List<Map<String, dynamic>>; // 给复杂的 Map 类型起名
  2. typedef IntBinaryOp = int Function(int a, int b); // 给复杂的函数起别名
  3. typedef IntOp = int Function(int a, int b); // 函数类型别名（typedef），申明一个函数签名，一般做函数传参限制

class VariableDemo {
  // 1️⃣ 普通变量
  int a = 10;                  // 显式类型，非空，类型固定
  var b = 20;                  // 自动类型推导，编译后类型固定为int
  var a = null;                // 编译后，类型会进行推断 固定为dynamic
  dynamic c = "hello";         // 动态类型，可以动态赋值其他类型
  Object d = 30;               // 非空指定类型
  Object? e;                   // 可空指定类型
  int? f;                       // 可空类型

  // 2️⃣ final / const 都支持自动推断类型，所以可以省略显示的类型声明
  final int g = 100;           // 运行时常量，只能赋值一次
  final h = "final var";       // 自动推导类型
  late final int i;            // 延迟初始化，只能赋值一次
  const int j = 200;           // 编译时常量

  // 3️⃣ 集合变量
  List<int> list1 = [1, 2, 3];          // List
  var list2 = <String>["a", "b"];       // 自动类型推导
  final List<int> list3 = [4, 5, 6];    // final 集合（集合不可重新赋值，但可修改元素）
  const List<int> list4 = [7, 8, 9];    // const 集合，编译时常量，也不可修改，增删集合内部元素

  // 4️⃣ 函数变量
  int Function(int, int) sum = (a, b) => a + b;  // 匿名函数赋值
  IntOp multiply = (a, b) => a * b;              // typedef 类型函数

  // 5️dart late 变量
  late int k;                     // 延迟初始化 非final 的使用 late，可以避免申明变量不赋值报错，可以赋值多次
  late final int l;               // 延迟初始化 + final， 只能延迟赋值一次

class FunctionDemo {
  int value;
  FunctionDemo(this.value);
  // 1️⃣ 普通方法
  int increment() {
    return value + 1;
  }
  // 2️⃣ 箭头函数（单表达式方法）
  int doubleValue() => value * 2;

  // 3️⃣ Getter / Setter
  int get squared => value * value;
  set squared(int val) => value = val ~/ value;

  // 4️⃣ 普通匿名函数 / 函数表达式
  var sum = (int a, int b) {
    return a + b;
  };

  // 5️⃣ 箭头匿名函数
  var triple = (int x) => x * 3;

  // 6️⃣ 异步方法
  Future<int> fetchValue() async {
    await Future.delayed(Duration(seconds: 1));
    return value;
  }

  Future<int> fetchValue2() async => value * 2;

  // 7️⃣ 同步生成器函数
  Iterable<int> countTo(int n) sync* {
    for (int i = 1; i <= n; i++) yield i;
  }

  // 8️⃣ 异步生成器函数
  Stream<int> asyncCount(int n) async* {
    for (int i = 1; i <= n; i++) yield i;
  }

  // 9. 高阶函数：接收函数
  void applyOperation(int a, int b, IntBinaryOp op) {
    print(op(a, b));
  }

  // 10. 高阶函数：返回函数
  Function multiplier(int x) {
    return (int y) => x * y;
  }
}

dart语言 `extends` vs `implements` vs `with`
 `extends`    继承class类暴露的成员函数和成员属性，子类可以重写（Override）父类方法，也可以直接使用父类实现，单继承
 `implements` 必须实现目标class类暴露的所有成员函数和成员变量（成员变了可以通过重新定义来实现，或者实现get/set方法实现两种办法）
 `with`       混入一个使用mixin定义的类，可以把混入的类中的成员函数和成员变量看做是这个类自己定义的（用于解决单继承的局限，提高代码拆分和复用）


# 五、集合（必问
dart 的 List / Set / Map 的区别？
List  有序、可重复 （与Java的 ArrayList在内存布局和基本算法上是高度一致的）
Set  无序、不重复 （实现原理类似 LinkedHashMap）
Map  key-value （实现原理类似HashMap）

---

dart List 的常用高阶方法？
map：将列表的每个元素映射为另一个值，返回一个新的迭代器对象（Iterable）。 var newList = nums.map((e) => e * 2).toList();
where：根据条件筛选元素，返回满足条件的子集合（Iterable） var subList = nums.where((e) => e % 2 == 0).toList();
fold： 累加 / 聚合列表元素，带一个初始值。var sum = nums.fold(0, (previous, element) => previous + element);
reduce：累加 / 聚合列表元素，没有初始值，使用列表第一个元素作为初始值。 var sum = nums.reduce((value, element) => value + element);
any：bool isOk = list.any((item) => item.type == 2);  判断列表是否有至少一个元素满足条件，返回 bool
every： bool isAllDone = list.every((item) => item.type == 2);  判断列表是否所有元素都满足条件，返回 bool

Dart 泛型是运行期还是编译期？
  Dart 的泛型是“具体化”的（Reified Generified），即运行期依然保留类型信息。
    var names = <String>['Alice', 'Bob'];
    // 在 Dart 运行期，这是合法的且返回 true
    print(names is List<String>); // true
    print(names is List<int>);    // false

## 22️⃣ Widget 为什么是不可变的？
1. flutter 源码 Widget被 @immutable修饰，被 @immutable 修饰的类及其子类的成员变量 都必须是final的 只能赋值一次，不可再改变。
2. Widget不可变是flutterUI体系的核心规则，为的是让应用层开发不再关注UI组件的中间状态，也就是View复杂的生命周期变化。
3. 让应用层只需要通过配置不可改变的widget来构建UI，实现式UI构建，方便element树复用逻辑的diff 提高性能

flutter 为什么大量 Widget 构造函数是 const？
1. flutter中所有的widget都是不可变的 也就是成员变量都是final的， 那么widget的构造函数就可能使用const修饰（需要widget的所有final修饰的成员变量都在编译期确定才可以）
2. 有些final修饰的的成员变量 需要在运行期间才能确定，这就导致了部分widget的构造函数，不能使用const修饰
3. dart 语言的一个特性是 const修饰的构造函数，在创建对象的时候 也使用const修饰，如果参数都相同，就可以进行内存复用

---
flutter StatefulWidget 的 state 为什么单独拆？
答案： Widget 是不可变的配置 State 是可变数据 解耦生命周期
      State 的生命周期： State 实例在 Widget 重建时会被 保留（Retain）。只要 Widget 的 runtimeType 和 key 没有变化，
      Flutter 框架就会重用同一个 State 对象，并将新的 Widget 引用挂载到该 State 上。
