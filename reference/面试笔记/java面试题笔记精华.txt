ClassLoader:java类加载器，功能是把java编译后的字节码文件加载到内存当中。
classLoad加载java字节码文件，一般有7个流程：
1.加载：1. 把类加载到内存中，生产对应的类对象, 2.把静态储存结构转化为方法区运行时的数据结构？
2.验证：验证字节码文件是否符合对应版本的java虚拟机规范
3.准备：初始化静态变量，分配内存
4.解析：把常量池中的符号引用转化为直接引用（把分配好的内存地址和符号对应起来建立指向关系对）
5.初始化类的静态变量：
6.使用
7.卸载
Classloader类型从加载优先级类型分为：
启动类加载器（C层代码）： 最顶层，由 JVM 本身实现，加载 核心类库，如 java.lang.*、java.util.*、java.io.*
扩展类加载器（Java层代码）：   加载 JDK 扩展库（$JAVA_HOME/lib/ext 或模块系统中的扩展模块）
应用程序类加载器（Java层）：（App ClassLoader / System ClassLoader）默认加载 classpath 下的应用类，也是双亲委派的最后一级，父类是扩展类加载器
自定义ClassLoader
加载机制：双亲委托机制，优先看上级classLoader是否可以加载，不可以加载则自己加载，依次递归。（应用程序类加载器->扩展类加载器->启动类加载器）


JVM内存结构：
1. 方法区：最小16M，最大64M，储存类信息和静态变量信息，各个线程共享内存区域
          运行时常量池是方法区的一部分
2. java方法虚拟机栈：每个方法被执行的时候 都会创建一个“栈帧”用于存储局部变量表，声明周期与线程相同，是线程私有的
3. 本地方法栈：虚拟机栈基本类似，只为native方法
4. 程序计数器：是最小的一块内存区域，它的作用是当前线程所执行的字节码的行号指示器
5. 堆：一般new关键词创建对象所分配的内存区域叫做堆，堆是由所有线程共享的,所以需要加锁保证不冲突，
      因此性能没有栈性能好。
      堆内存分配策略优先级：栈->TLAB（Thread Local Allocation Buffer 也属于堆比较特殊）->年老代分配->eden分配
       JVM在Server模式下的逃逸分析可以分析出某个对象是否永远只在某个方法、线程的范围内->使用栈分配
       对象内存分配的两种方法：1.指针碰撞 2. 空闲列表
       哪种分配方式由Java堆是否规整决定，而Java堆是否规整又由所采用
       的垃圾收集器是否带有压缩整理功能决定。
  Java虚拟机的堆内存中，主要分为年轻代和老年代两个部分。而在年轻代中，又可以细分为Survivor区，Eden区、From区和To区：
  Eden区是年轻代中最大的一个区域，用于存放新创建的对象。当一个对象被创建时，首先会被分配到Eden区
  年轻代堆内存用来存放新近创建的对象：使用minorGC回收->年轻代对象在进过minorGC后可能进入老年代堆内存区域
  老年代堆内存主要存放应用程序中生命周期长的内存对象。->使用majorGC回收



  Java设计模式根据其解决的问题和应用场景，通常分为三大类：
  创建型模式（Creational Patterns）:单例模式, 工厂方法模式, 抽象工厂模式,建造者模式,原型模式
  结构型模式（Structural Patterns）:适配器模式, 桥接模式, 组合模式, 装饰器模式, 外观模式, 享元模式, 代理模式
  行为型模式（Behavioral Patterns）:职责链模式, 命令模式 解释器模式 迭代器模式 中介者模式 备忘录模式 观察者模式 状态模式 策略模式 模板方法模式 访问者模式
  每一类模式侧重于解决特定类型的问题，并提供不同的方法来设计和实现灵活且可扩展的系统。

java中的四种引用： 1.强引用（不会回收-进程宁愿内存溢出也不会回收）
                2.软引用（内存不够回收，比如图片缓存-弱引用不会影响对象是否被 GC 回收）
                3.弱引用（GC 的时候回收-防止内存泄漏的监听器，弱引用不会影响对象是否被 GC 回收）
                4.虚引用（不能获取引用对象，做内存回收后续清理和通知）

Java死锁： 定义：两个或两个以上的线程在执行过程中因为相互等待锁，但是又拿不到锁导致程序卡死，无法正常执行。
          发生的情形：线程1持有锁1但是还需要锁2才能执行下去，线程2持有锁2但是需要锁1才能执行下去。

悲观锁：假设线程每次要执行需要加锁的同步代码的时候，这段代码大概率被其他线程正在执行，所以为了同步安全这段代码必须加锁。
乐观锁：假设线程每次要执行需要加锁的同步代码的时候，这段代码大概率没有其他线程在执行，所以可以不加锁，只在提交时候检测冲突。
      悲观锁和乐观锁不是实际的API锁，而是描述了两种需要加锁同步的代码的多线程并发情况。

synchronized 和 ReentrantLock
synchronized：用于同步代码块或同步方法。特性：1.由 JVM 隐式管理线程对锁的获取与释放。2. 不可中断（无法响应 Thread.interrupt()） 3. 非公平，无法保证等待时间最长的线程优先获得锁
ReentrantLock(重入锁)：需要程序员显示的调用api进行线程对锁的获取和释放。特性：1. 支持锁公平和非公平机制的选则 2.可以响应线程中断

Java注解： 用于给Java的类， 成员函数， 成员变量标记相关信息，使程序在编译过程或者运行过程中可以获取这些信息进行相关操作。
          Java中 使用@interface进行注解类的定义

Java反射： Java程序在运行过程中可以动态获取任何一个类的全部信息（类中定义的所有函数和成员变量，泛型，注解），并且可以调用这些函数和变量。

String ，StringBuilder, StringBuffer 区别：
 String：线程安全，不可变对象，每次拼接都会创建新的String对象，浪费内存
 StringBuilder： 线程不安全，可变对象，拼接字符串不会创建新的对象
 StringBuffer： 线程安全，可变对象，拼接字符串不会创建新的对象

 Java中的浅拷贝和深拷贝：
 浅拷贝： 浅拷贝通常通过实现 Cloneable 接口并重写 Object.clone() 方法实现，会建一个新对象
         新对象内部引用的成员变量，如果是引用类型，依然指向原始对象的内存地址，所以如果原始对象的引用类型对象内的数据改变了，浅拷贝的对象的相应成员数据也会改变。
         新对象内部引用的成员变量，如果是基本数据类型（int, boolean 等）会被值拷贝，有自己独立的副本。
 深拷贝：深拷贝会递归地拷贝对象及其引用的所有成员，确保拷贝后的对象与原对象完全独立，互不影响。

 okhttp：为什么使用拦截器模式，里面的那么多拦截器分别负责哪些功能
         解耦了各个阶段的网络逻辑，如重试、缓存、建立连接。逻辑解耦与单一职责，高度的可扩展性，流程的可观测性
         RetryAndFollowUpInterceptor：重试与重定向。负责处理网络故障后的自动重试，以及 HTTP 3xx 状态码的自动跳转
         BridgeInterceptor：应用层与网络层桥接。将用户构建的 Request 转换为标准的 HTTP 数据包
         CacheInterceptor：缓存http协议中的缓存协议。Cache-Control Etag/Last-Modified）
                           http 中 的缓存协议 可以优化网络请求路径，如果本地有完善可以使用的缓存，就直接从本地获取，如果CDN有可以使用的有效缓存，就不会继续向中心服务器请求。
         ConnectInterceptor：建立连接处理进行TCP/TLS 三次握手逻辑，查找或创建可用的 RealConnection，并获取 HttpCodec（用于编码解码)
         CallServerInterceptor：数据读写。真正向 Socket 写入 Request body 字节流，并从 Socket 读取 Response 数据

 TCP的三次握手和四次挥手：
  三次握手：
     1.客户端发送建立链接的请求信号（SYN）：
     2.服务端收到客户端的建立链接的请求，并且回复客户端（SYN + ACK）表示可以正常建立链接
     3.客户端向服务器发送确认建立链接的请求（ACK）同时可能携带应用层的请求报文。
  四次挥手：
     1. 客户端向服务端发送(不需要继续向服务端发送业务数据的)请求信号FIN
     2. 服务端收到客户端发生的（不写）信号后：给客户端发送确认（客户端不写的）信号ACK
     3. 服务端再次给客户端发送可以关闭链接的信号FIN
     4.客户端收到服务端第1个信号会关闭write的链接，客户端收到服务端第2个信号会关闭read链接ACK
